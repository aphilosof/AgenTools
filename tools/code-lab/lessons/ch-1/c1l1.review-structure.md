## Findings (fixes)

1. **Dangling `[[sequence]]` glossary link in the warm-up (minor).** The warm-up
   links `[[sequence]]`, but there is no `sequence` glossary entry in this lesson
   and Ch.0 is not yet authored as data, so no glossary hover target exists; the
   link currently dangles. The brief's warm-up spec asks only to *recall the idea*
   of "a sequence of exact steps" in prose, not to add a `[[term]]` link. Fix:
   drop the `[[ ]]` around `sequence` (plain text) until Ch.0 defines the term, or
   add a `sequence` glossary entry here. Everything else linked resolves.

2. **Two concept paragraphs score below the grade 7–9 band (minor / judgment
   call).** Flesch-Kincaid (harness formula, code stripped): warm-up 7.3, S1 6.2,
   S2 5.7, S3 8.6, S4 8.0, recap 7.3, combined 6.8. The band is a *ceiling* per
   AUTHORING.md ("never push prose down toward baby-talk"), so nothing exceeds it;
   but S1 (6.2) and S2 (5.7) sit under the stated 7–9 floor. FK is approximate and
   these are the gentlest intro beats, so this is not a blocker — flagging that the
   two earliest concept paragraphs read a notch simpler than the lesson's target,
   in case the author wants to firm them up. No paragraph is too hard.

3. **Test-harness vs. current-schema mismatch — the "passes the harness" claim is
   not verifiable as-is (important, but a harness/environment defect, not a draft
   defect).** `tests/run.js` validates an *older* schema generation than this draft
   (and `schema.md`) use:
   - It requires `l.chapter` (draft/schema use `chapter` + `lessonNo`),
   - requires a single-object `l.codex` with `topic/pattern/note` (draft/schema use
     a **codex array**),
   - and the reading-level check (invariant 6) grades `l.explain` with
     `MAX_GRADE = 7` — but the draft has **no `explain` field** (prose lives in
     `warmup`, `subsections[].concept.md`, `recap.md`), so the check does
     `if (!isNonEmptyString(l.explain)) continue` and **silently skips** the
     lesson; the prose is never graded by the harness. The harness comment even
     says "grade 6 reading," conflicting with schema.md's grade 7–9.

   Consequence: `npm test` cannot currently validate this draft against the
   current schema. The draft's header asserts conformance to the harness; that
   assertion can't be confirmed until `tests/run.js` is updated to the
   `chapter/lessonNo` + `subsections` + `codex[]` + grade-7–9 schema (and to read
   prose from `warmup`/`concept.md`/`recap.md`). This is a build-task gap to raise,
   not a lesson edit. All invariants were instead verified manually above and pass.
